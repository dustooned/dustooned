# Journal System

Journal posts are MDX files in `src/content/journal/`.

Required fields:

- `title`
- `date`
- `excerpt`

Optional fields:

- `thumbnail`
- `tags`
- `featured`

Use `RawPostInsert.astro` for intentionally custom inserted blocks.

Do not execute arbitrary external scripts by default. Iframes and embeds should be reviewed before publishing.

## Thumbnails

`thumbnail` is normally a local `thumb.webp` generated by
`node scripts/generate-smart-thumbnails.mjs` from an `image-1.*` source file in the post's
`public/images/journal/{slug}/` folder — see `docs/ASSET_GUIDE.md` for the generator and the
animated-GIF gotcha (a GIF-sourced post must regenerate its thumb as an animated webp, not a
frozen frame, since `JournalLayout.astro` reuses `thumbnail` as the large hero image at the
top of the post itself, not just the grid card).

For posts whose content *is* a video embed (`RawPostInsert` + YouTube iframe) rather than a
local image, `thumbnail` can instead point straight at the YouTube thumbnail, same convention
as Motion:

```yaml
thumbnail: "https://img.youtube.com/vi/{videoId}/hqdefault.jpg"
```

No local asset needed — `withBase()` passes absolute `https://` URLs through unchanged.

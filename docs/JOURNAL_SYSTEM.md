# Journal System

Journal posts are MDX files in `src/content/journal/`.

Required fields:

- `title`
- `date`
- `excerpt`

Optional fields:

- `thumbnail`
- `image`
- `tags`
- `featured`

Use `RawPostInsert.astro` for intentionally custom inserted blocks.

Do not execute arbitrary external scripts by default. Iframes and embeds should be reviewed before publishing.

## Thumbnails vs. hero image

Two separate fields cover two separate jobs — do not conflate them:

- **`thumbnail`** — small, cropped `thumb.webp` used only in grid cards (`/journal` list and
  any other place a `JournalCard` renders). Generated by
  `node scripts/generate-smart-thumbnails.mjs` from an `image-1.*` source file in the post's
  `public/images/journal/{slug}/` folder — see `docs/ASSET_GUIDE.md` for the generator.
- **`image`** — the full original source file (same `image-1.*`, unresized/uncropped), used
  as the large hero at the top of the post itself in `JournalLayout.astro`. The generator
  writes this automatically alongside `thumbnail` — you don't need to set it by hand for
  posts backed by a local image. `JournalLayout` falls back to `thumbnail` if `image` isn't
  set (e.g. video-only posts, see below).

Regenerating via the script keeps both in sync; don't hand-edit `thumbnail` without also
checking `image` still points at the right source.

### Gotcha: GIF thumbnails must stay animated, but small

GIF-sourced thumbnails are generated at a much smaller size than static ones (640x480 vs.
1600x1200, lower quality) specifically because animated webp cost scales with pixel count
per frame — the original 1600x1200 animated thumbnails were multi-megabyte files that visibly
lagged the `/journal` grid. `image` (the post hero) still uses the original, unresized
source file — that one only loads once per post view, not repeated across a grid, so its
larger size is fine. See `docs/ASSET_GUIDE.md` for the exact dimensions/quality constants.

For posts whose content *is* a video embed (`RawPostInsert` + YouTube iframe) rather than a
local image, `thumbnail` can instead point straight at the YouTube thumbnail, same convention
as Motion — no `image` field needed, no local asset generated:

```yaml
thumbnail: "https://img.youtube.com/vi/{videoId}/hqdefault.jpg"
```

`withBase()` passes absolute `https://` URLs through unchanged.
